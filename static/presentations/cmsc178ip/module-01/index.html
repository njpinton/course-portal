What is Digital Image Processing?

Definition:

The use of computer algorithms to perform image processing on digital images.

  • Input: Digital image(s)
  • Output: Enhanced image or extracted information

Key Applications:

  • Medical Imaging (CT, MRI)
  • Satellite & Remote Sensing
  • Autonomous Vehicles
  • Face Recognition
  • Social Media Filters

Digital Image Fundamentals

The mathematical representation of images

Image as a 2D Function

Continuous Image:

\( f(x, y) \)

where:

  • \( x, y \) are spatial coordinates
  • \( f \) is the intensity (brightness)

Digital Image:

\( f[m, n] \)

where:

  • \( m, n \) are discrete indices
  • \( m \in \{0, 1, ..., M-1\} \)
  • \( n \in \{0, 1, ..., N-1\} \)

From Continuous to Discrete

Sampling creates a grid of pixels, each with a discrete intensity value

Image Coordinate System

Origin at top-left; x increases right, y increases down

Spatial Sampling

Sampling: The process of converting continuous spatial coordinates into discrete pixel locations.

  • Higher sampling rate = more pixels = more detail
  • Lower sampling rate = fewer pixels = aliasing artifacts

\( \text{Nyquist Theorem: } f_s \geq 2 \cdot f_{max} \)

Effect of Sampling Rate

Lower sampling rates lose fine details and introduce aliasing

Spatial Resolution

Higher resolution preserves more detail but requires more storage

Intensity Quantization

Quantization: The process of mapping continuous intensity values to discrete levels.

  • 8-bit: 256 levels (0-255)
  • 4-bit: 16 levels
  • 1-bit: 2 levels (binary)

\( L = 2^k \text{ levels, where } k = \text{bits per pixel} \)

Effect of Quantization

Fewer bits = visible banding (posterization effect)

Image Types

Binary, Grayscale, and Color

Common Image Types

Binary Images (1-bit)

  • Only two values: 0 (black) or 1 (white)
  • 1 bit per pixel
  • Used for: Documents, masks, morphological operations
# Thresholding to binary
binary = (gray > 128).astype(np.uint8) * 255

Memory:

\( \text{Size} = \frac{W \times H}{8} \text{ bytes} \)

Example: 640x480 = 38.4 KB

Grayscale Images (8-bit)

  • 256 intensity levels (0-255)
  • 1 byte per pixel
  • 0 = black, 255 = white
# RGB to Grayscale (luminosity)
gray = 0.299*R + 0.587*G + 0.114*B

Memory:

\( \text{Size} = W \times H \text{ bytes} \)

Example: 640x480 = 307.2 KB

Color Images (24-bit RGB)

  • 3 channels: Red, Green, Blue
  • 8 bits per channel = 24 bits total
  • 16.7 million possible colors
# Access RGB channels
R = image[:, :, 0]
G = image[:, :, 1]
B = image[:, :, 2]

Memory:

\( \text{Size} = W \times H \times 3 \text{ bytes} \)

Example: 640x480 = 921.6 KB

Bit Depth Comparison

Higher bit depth = smoother gradients, larger files

Color Models

RGB, HSV, and Color Spaces

RGB Color Model

RGB: Additive Color Mixing

  • Red + Green = Yellow
  • Green + Blue = Cyan
  • Red + Blue = Magenta
  • R + G + B = White
# Pixel color tuple
pixel = (R, G, B)  # Each 0-255

# Pure colors
red = (255, 0, 0)
green = (0, 255, 0)
blue = (0, 0, 255)
white = (255, 255, 255)
black = (0, 0, 0)

HSV Color Model

HSV: More Intuitive for Humans

  • H (Hue): Color type (0-360deg)
  • S (Saturation): Color purity (0-100%)
  • V (Value): Brightness (0-100%)

Advantages:

  • Intuitive color selection
  • Better for thresholding
  • Separates luminance from color
import cv2

# Convert BGR to HSV
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# Extract channels
H = hsv[:, :, 0]  # 0-180 in OpenCV
S = hsv[:, :, 1]  # 0-255
V = hsv[:, :, 2]  # 0-255

Memory Calculation

Uncompressed Image Size:

\( \text{Bytes} = \text{Width} \times \text{Height} \times \frac{\text{Bits per Pixel}}{8} \)


Examples:

  • 1920x1080 grayscale: 1920 x 1080 x 1 = 2.07 MB
  • 1920x1080 RGB: 1920 x 1080 x 3 = 6.22 MB
  • 4K RGB: 3840 x 2160 x 3 = 24.88 MB

Summary

Key Takeaways

  1. Digital images are 2D arrays of discrete pixel values
  2. Sampling determines spatial resolution (pixels)
  3. Quantization determines intensity levels (bit depth)
  4. Image types: Binary (1-bit), Grayscale (8-bit), Color (24-bit)
  5. Color models: RGB (additive) vs HSV (intuitive)
  6. Trade-off: Higher quality = more storage
