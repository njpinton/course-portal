Image Types

Understanding digital image representations

What is a Digital Image?

A digital image is a 2D array of pixels, where each pixel contains discrete intensity values representing color or brightness at specific spatial locations.

Mathematical Foundation:

  • Image function: \( f(x, y) \)
  • Intensity levels: \( L = 2^k \)
  • Pixel values: \( 0 \leq f(x,y) \leq L-1 \)

File Size Formula:

\( \text{Size} = M \times N \times \frac{\text{bits}}{8} \)

Four Main Image Categories

Binary Images (1-bit)

Only two intensity values: 0 (black) and 1 (white). Each pixel requires 1 bit.

  • Pixel values: \( f(x,y) \in \{0, 1\} \)
  • Storage: \( \frac{M \times N}{8} \) bytes
  • 8:1 memory efficiency vs grayscale

Applications:

  • OCR systems
  • Barcode reading
  • Medical bone analysis
  • Industrial inspection

Creation: Thresholding, dithering, edge detection

Grayscale Images (8-16 bit)

  • 8-bit: 0-255 (standard)
  • 12-bit: 0-4095 (cameras)
  • 16-bit: 0-65535 (scientific)

RGB Conversion:

\( Y = 0.299R + 0.587G + 0.114B \)

Color Images - RGB Model

RGB uses additive color mixing with 8 bits per channel = 24-bit color.

  • Each channel: 0-255
  • Total colors: \( 256^3 = 16,777,216 \)
  • Storage: \( 3 \times M \times N \) bytes

Extended Formats:

  • RGB565: Mobile (16-bit)
  • ARGB8888: 32-bit with alpha

Alternative Color Models

CMYK (Subtractive):

  • Cyan, Magenta, Yellow, Black
  • Used for printing

HSV (Perceptual):

  • Hue: 0-360deg
  • Saturation: 0-100%
  • Value: 0-100%

Model Selection:

  • Display: RGB
  • Print: CMYK
  • Perception: HSV
  • Compression: YUV

Indexed Color & Palette Systems

Uses a Color Lookup Table (CLUT) with up to 256 colors. Each pixel stores an index to the palette.

  • Pixel values: 0-255 (indices)
  • Storage: \( M \times N + 768 \) bytes
  • Algorithms: Median Cut, Octree, K-means

Applications: GIF, retro games, scientific visualization

Trade-off: Reduced size vs limited colors

Storage Formats

BMP, JPEG, PNG, GIF, TIFF

Image Storage Formats - Overview

Formats define how pixel data is organized, encoded, and stored. They determine compression, metadata, and compatibility.

Classification:

  • Raster vs Vector
  • Compressed vs Uncompressed
  • Lossless vs Lossy

Categories:

  • Uncompressed: BMP, PPM
  • Lossless: PNG, GIF, TIFF
  • Lossy: JPEG, WebP

Storage Formats - File Size Comparison

BMP Format - Windows Bitmap

Microsoft's native raster format with uncompressed pixel data.

File Structure:

  • File Header: 14 bytes
  • Info Header: 40+ bytes
  • Color Palette (optional)
  • Pixel Data: bottom-to-top

Advantages:

  • Simple structure
  • No compression artifacts
  • Fast loading

Disadvantages:

  • Very large file sizes
  • No transparency

JPEG Format - Lossy Compression

Uses DCT-based lossy compression optimized for photographs with smooth color transitions.

Compression Pipeline:

  1. RGB to YCbCr conversion
  2. Chrominance subsampling
  3. 8x8 DCT transform
  4. Quantization
  5. Huffman coding

Quality Control:

  • Quality: 1-100 scale
  • Ratio: 10:1 to 50:1
  • Artifacts: Blocking, ringing

JPEG Quality Comparison

JPEG Compression Artifacts

  • Blocking: 8x8 DCT block boundaries visible
  • Ringing: Oscillations near sharp edges
  • Color Bleeding: Chrominance subsampling effects

JPEG Compression Procedure

PNG Format - Lossless Compression

Lossless compression with transparency and gamma correction. Designed as GIF replacement.

Specifications:

  • Color: Grayscale, RGB, Palette, Alpha
  • Bit Depths: 1, 2, 4, 8, 16 bits
  • Compression: DEFLATE (LZ77 + Huffman)
  • Interlacing: Adam7 progressive

Advantages:

  • Lossless compression
  • Full alpha transparency
  • Error detection (CRC)

Disadvantages:

  • Larger than JPEG for photos
  • No animation

PNG Compression Procedure

GIF & TIFF Formats

GIF - Graphics Interchange Format

  • 256 colors per frame
  • LZW compression
  • Animation support
  • 1-bit transparency

TIFF - Tagged Image File Format

  • Multiple compression options
  • 1-bit to 32-bit per channel
  • Extensive metadata
  • Multi-page support

Use cases: GIF for animations/logos; TIFF for professional/medical imaging

Compression

Fundamentals & Techniques

Compression Fundamentals

Compression reduces file sizes by eliminating redundancy and exploiting perceptual limitations of human vision.

Types of Redundancy:

  • Spatial: Neighboring pixels similar
  • Spectral: Color channels correlated
  • Temporal: Video frame similarities
  • Psychovisual: Human eye limitations

Performance Metrics:

  • Compression Ratio: Original / Compressed
  • Bit Rate: Bits per pixel (bpp)
  • PSNR: Peak Signal-to-Noise Ratio
  • SSIM: Structural Similarity

Lossless Compression

Achieves perfect reconstruction by exploiting statistical redundancy without discarding information.

Huffman Coding:

  • Variable-length codes
  • Frequency-based assignment
  • Used in JPEG, PNG

LZW:

  • Dictionary-based patterns
  • Used in GIF, TIFF

RLE (Run-Length Encoding):

  • Replace repeated values
  • Example: AAABBBBCC → 3A4B2C
  • Great for binary images

DEFLATE:

  • LZ77 + Huffman
  • Used in PNG

Huffman Encoding

  • Optimal: Minimizes average code length
  • Variable Length: Frequent symbols get shorter codes
  • Prefix-Free: No code is prefix of another

Huffman Encoding Procedure

Run-Length Encoding (RLE)

  • Binary Images: Excellent compression for text, line art
  • Simple Graphics: Images with large uniform areas
  • Fax Machines: Standard for document transmission

Lossy Compression

Achieves high compression ratios by permanently removing information with minimal perceptual impact.

Transform-Based (DCT):

  • JPEG standard
  • Spatial to frequency domain
  • 8x8 block processing
  • Quantize high frequencies

Wavelet-Based:

  • JPEG 2000
  • Multi-resolution analysis
  • Less blocking artifacts
  • Medical, satellite imagery

DCT - JPEG Foundation

  • Spatial → Frequency: Transform 8x8 blocks
  • Energy Compaction: Most energy in low frequencies (top-left)
  • Quantization: Remove high-frequency details

Advanced Compression Standards

Next-Generation Formats:

  • WebP: Google's VP8-based
  • HEIF: H.265-based
  • AVIF: AV1-based open standard
  • JPEG XL: Latest JPEG standard

Performance Comparison:

  • Compression: AVIF > HEIF > WebP > JPEG
  • Encode Speed: JPEG > WebP > HEIF
  • Browser Support: JPEG > WebP > AVIF

Future: AI-based compression, perceptual metrics

Real-World Applications

Industry Case Studies

Industry Applications

Medical Imaging:

  • DICOM: JPEG-LS, JPEG 2000 lossless
  • No diagnostic information loss
  • 16-bit CT scans, 12-bit X-rays

Social Media:

  • Facebook: WebP, HEIF
  • Instagram: Aggressive JPEG + AI
  • Billions of images daily

E-commerce:

  • Multi-format delivery
  • Progressive loading
  • Responsive images

Scientific/Satellite:

  • NASA: Custom wavelets
  • Google Earth: Multi-resolution
  • Lossless compression required

Summary

Key Takeaways

  1. Image Types: Binary, grayscale, color, indexed representations
  2. Storage Formats: BMP (uncompressed), JPEG (lossy), PNG (lossless)
  3. Color Models: RGB for display, CMYK for print, HSV for perception
  4. Lossless Compression: Huffman, LZW, RLE - perfect reconstruction
  5. Lossy Compression: DCT, wavelets - trade quality for size
  6. Trade-off: Quality vs file size depends on application

Practical Guidelines

Format Selection

  • Photos: JPEG for sharing, RAW for archival
  • Graphics: PNG for transparency
  • Web: WebP/AVIF for modern browsers
  • Medical: Always lossless

Decision Framework

  • Quality needs?
  • Storage constraints?
  • Processing power?
  • Compatibility required?
